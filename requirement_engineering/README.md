# Requirements Engineering — Automotive

This folder covers requirements engineering practices, tools, and workflows used in automotive ECU and ADAS development.

## Files

| File | Description |
|------|-------------|
| `01_requirements_engineering_guide.md` | Complete guide: EARS notation, DOORS, Polarion, traceability, STAR scenarios, interview Q&A |
| `02_doors_practical_guide.md` | IBM DOORS practical guide: modules, DXL scripting, baselines, change proposals |

## Quick Reference

- **EARS Patterns**: Ubiquitous, Event-driven, State-driven, Optional, Unwanted behaviour
- **Traceability Chain**: Stakeholder → System → SW/HW → Module → Test
- **Key Tools**: IBM DOORS Next, Polarion ALM, JIRA + Requirements plugin, Enterprise Architect
- **Review Standard**: ISO/IEC/IEEE 29148 | Automotive: ASPICE SYS.2, SWE.1

## Applicability

- ASPICE Level 2/3 compliance projects
- ISO 26262 Functional Safety (ASIL A–D)
- ADAS feature development (AEB, LKA, ACC)
- Multi-ECU integration programs
