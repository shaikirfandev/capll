### How to Use These Scenarios

Each scenario follows the same format:
1. **Interviewer question** — the exact style of question asked in interviews
2. **Investigation path** — systematic, layer-by-layer approach
3. **CAPL / Python / ADB code** — where relevant, actual code you could write
4. **Test cases** — formal test cases derived from the scenario
5. **Root cause summary** — what the bug most likely is

### Key Interview Principle
Always structure your answer as:
> "First I would isolate which layer the fault is in, then I would use [specific tool] to confirm, and here is what I expect to see..."

Never start with: *"It must be a software bug"* without evidence.
